# Report Templates Reference

Output templates for each depth mode. Adapt sections as needed based on the topic.

## Quick Mode Template

```markdown
# [Topic]: Quick Research Summary

*Date: [YYYY-MM-DD] | Sources: [N] | Mode: Quick*

## Summary

[3–5 sentence overview of key findings]

## Key Findings

- **[Finding 1]**: [Brief explanation with data] [1]
- **[Finding 2]**: [Brief explanation with data] [2]
- **[Finding 3]**: [Brief explanation with data] [3]

## Sources

[1] [Author/Org]. "[Title]." [Publication], [Date]. [URL]
[2] ...
```

## Practical Decision Aid Templates

Include these elements where the topic calls for them. They supplement — not replace — the main report structure.

### Quantitative Scoring Matrix (for comparison/selection topics)

```markdown
## 量化决策矩阵 / Decision Matrix

| Factor | Weight | Option A Score | Option B Score |
|--------|--------|---------------|---------------|
| [Factor 1] | [%] | [1-5] | [1-5] |
| [Factor 2] | [%] | [1-5] | [1-5] |
| [Factor 3] | [%] | [1-5] | [1-5] |
| **Weighted Total** | 100% | **[X.X]** | **[X.X]** |

*Adjust weights based on project priorities. Scores derived from research evidence.*
```

### Scenario-Based Recommendation Table (for technology/strategy selection)

```markdown
## 场景化选型建议 / Scenario Recommendations

| Scenario | Recommendation | Rationale |
|----------|---------------|-----------|
| [Scenario 1: e.g., startup MVP] | [Option A] | [1-sentence reason with citation] |
| [Scenario 2: e.g., high-traffic infra] | [Option B] | [1-sentence reason with citation] |
| [Scenario 3: e.g., hybrid approach] | [A + B] | [1-sentence reason with citation] |
```

### Architecture / Relationship Diagram (for system/architecture topics)

```markdown
## 架构示意 / Architecture Overview

┌─────────────┐     ┌──────────────┐
│  [Layer 1]  │────▶│  [Layer 2]   │
│  (Tool A)   │     │  (Tool B)    │
└──────┬──────┘     └──────┬───────┘
       │                   │
       └───────┬───────────┘
               ▼
       ┌───────────────┐
       │  [Layer 3]    │
       │  (Tool C)     │
       └───────────────┘
```

### SWOT Table (for market/strategy topics)

```markdown
## SWOT 分析

| Strengths [N] | Weaknesses [N] |
|---------------|----------------|
| [S1] | [W1] |
| [S2] | [W2] |

| Opportunities [N] | Threats [N] |
|-------------------|-------------|
| [O1] | [T1] |
| [O2] | [T2] |
```

Use whichever aid best serves the reader's decision-making needs. Multiple aids can be combined in a single report.

## Continuation State Template

When a Deep mode report exceeds output limits and requires auto-continuation:

```markdown
<!-- CONTINUATION STATE — DO NOT INCLUDE IN FINAL REPORT -->
## Continuation Marker

### Progress
- Sections completed: [list]
- Sections remaining: [list]
- Current word count: [N]
- Target word count: [M]

### Citations Used
[1] through [N] — see bibliography in saved file

### Quality Metrics
- Sources consulted: [N]
- Tier 1–2 sources: [N]
- Cross-referenced claims: [N]

### Next Steps
1. Write section: [section name]
2. Write section: [section name]
3. Complete bibliography
4. Run quality gate checklist

### Instructions for Continuation Agent
Read the saved report file at [path]. Continue from section [N].
Maintain the same writing style, citation format, and depth level.
Do not repeat information already covered. Add new citations starting from [N+1].
```
