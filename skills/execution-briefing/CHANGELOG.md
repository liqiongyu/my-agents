# Changelog

All notable changes to this skill will be documented in this file.
This project adheres to [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) and [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

## [0.2.0] - 2026-03-29

### Added
- An optional quick briefing lane for compact, already-stable shaped issues that
  still need explicit checks, evidence, and escalation triggers.
- An approval-checkpoint variant for producing a brief while intentionally
  holding execution pending sign-off.
- A machine-readable brief-contract example for future orchestration or
  programmatic consumption.

### Changed
- Strengthened the core skill and methodology docs with explicit gate-source
  visibility, operational source-of-truth guidance for downstream review
  surfaces, and deeper research-backed boundary rationale.
- Expanded output templates and eval fixtures to cover quick-briefing,
  approval-hold behavior, and gate-source-of-truth expectations.

## [0.1.0] - 2026-03-29

### Added
- Initial `execution-briefing` skill focused on translating shaped issues into
  run-ready execution briefs and initial done contracts.
- A manual-first workflow with explicit boundaries against issue intake,
  issue-shaping, backlog admission, clarify, direct execution, and deep
  implementation planning.
- Methodology and output-template references covering authority hierarchy,
  run-level contract design, verification expectations, escalation triggers, and
  route-back notes.
- Lightweight trigger and qualitative eval fixtures plus projection settings for
  cross-platform installs.
