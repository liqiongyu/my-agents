# README Templates: Community Projects

Use this file for awesome lists, organization profile repos, and broader resource hubs. These READMEs should optimize for navigation, curation clarity, and contribution flow.

## Awesome List / Curated Collection

Use when the repo is a browsable list of tools, links, articles, or community resources.

Recommended order:
- Title, one-line scope statement, and a few legitimacy badges
- Table of contents for major categories
- Category sections with consistently formatted entries
- Articles/talks or related reading if relevant
- Brief contribution path and license link

Must not miss:
- Clear inclusion standards
- Consistent item formatting such as `[Name](link) — Description`
- A short contribution summary that links to the full guide

Micro skeleton:

```markdown
# Awesome Topic Name
## Contents
## Category A
## Category B
## Articles & Talks
## Contributing
## License
```

## Organization Profile (.github repo)

Use when the repo represents a GitHub organization and introduces its projects and community standards.

Recommended order:
- Organization name and mission statement
- About us or what the organization builds
- Key projects or starting points
- Getting involved
- Community standards and contact links

Must not miss:
- Links to the most important active projects
- A clear newcomer path
- Link-first references to code of conduct, contributing, and security docs

Micro skeleton:

```markdown
# Organization Name
## About Us
## Our Projects
## Getting Involved
## Community Standards
## Contact
```

## Resource Hub / Topic Collection

Use when the repo gathers resources around a topic but is not a strict awesome-list.

Recommended order:
- Title and audience statement
- Why this collection exists
- How to use it or recommended reading order
- Main sections grouped by level, topic, or use case
- Brief contribution path and license link

Must not miss:
- Orientation for beginners versus advanced readers
- Clear grouping logic for the resource list
- Freshness cues when links can go stale

Micro skeleton:

```markdown
# Topic Name Resource Collection
## Why This Exists
## How to Use This
## Contents
## Contributing
## License
```

## Community-Specific Writing Tips

- Navigation quality matters more than prose volume.
- Keep contribution guidance visible but short; link to the full policy file.
- Prefer consistent entry formatting over clever styling.
- If freshness matters, mention how links or submissions are reviewed.
