# README Templates: Content & Education Projects

Use this file for documentation sites, tutorials/courses, and blogs or content collections. Keep these READMEs navigation-first: the README should help readers find the right entry point quickly, not mirror the entire content tree.

## Documentation / Knowledge Base

Use when the repo primarily hosts docs, specs, handbooks, or a docs site.

Recommended order:
- Title and one-line explanation of what the docs cover
- What's inside or who the docs are for
- Quick navigation table linking first-stop pages
- Local preview instructions only if the docs are meant to be run locally
- Structure or authoring notes if contributors need orientation
- Contributing link and license link

Must not miss:
- A clear "start here" pointer for first-time readers
- Links to generated/live docs when they exist
- Link-first contributing guidance rather than copied policy text

Micro skeleton:

```markdown
# Project Docs
## What's Inside
## Quick Navigation
## Reading Locally
## How This Is Organized
## Contributing
```

## Tutorial / Course

Use when the repo teaches a topic progressively through modules, chapters, or exercises.

Recommended order:
- Title and learning outcome
- Prerequisites and target skill level
- Curriculum or module map
- Getting started and setup
- How to use the materials: order, exercises, solutions, projects
- Related resources, contributing link, and license link

Must not miss:
- A clear progression path
- Honest prerequisite guidance
- Whether solutions are included and how learners should use them

Micro skeleton:

```markdown
# Course Title
## What You'll Learn
## Prerequisites
## Curriculum
## Getting Started
## How to Use This
## Related Resources
```

## Blog / Content Collection

Use when the repo backs a writing site, newsletter archive, essay collection, or post repository.

Recommended order:
- Title and editorial focus
- About or what readers can expect
- Recent posts or topic map
- Local preview instructions only if contributors need them
- Writing/deployment notes when the repo is an active publishing source
- License link

Must not miss:
- The live site URL when one exists
- Topic or category navigation for larger archives
- Whether the repo is actively published or mostly archival

Micro skeleton:

```markdown
# Publication Name
## About
## Recent Posts
## Topics
## Running Locally
## Writing a New Post
## Deployment
```
