## Lean Canvas

Best for: Idea stage, startups, rapid iteration. A one-page business plan that forces focus.

```
+------------------+------------------+------------------+
|  PROBLEM         |  SOLUTION        |  UNIQUE VALUE    |
|  Top 3 problems  |  Top 3 features  |  PROPOSITION     |
|                  |                  |  Single, clear   |
|                  |                  |  message         |
+------------------+------------------+------------------+
|  KEY METRICS     |  UNFAIR          |  CHANNELS        |
|  Key numbers     |  ADVANTAGE       |  Path to         |
|  that tell you   |  Something that  |  customers       |
|  business is     |  cannot be       |                  |
|  working         |  easily copied   |                  |
+------------------+------------------+------------------+
|  COST STRUCTURE                     |  REVENUE STREAMS |
|  Customer acquisition costs         |  Revenue model   |
|  Distribution costs                 |  Lifetime value  |
|  Hosting / infrastructure           |  Revenue         |
|  People                             |  Gross margin    |
+-------------------------------------+------------------+
```

### How to Fill It

1. Start with **Problem** — if you cannot articulate 3 real problems, the business idea needs more validation.
2. **Customer Segments** — who has these problems most acutely? Be specific (not "everyone").
3. **Unique Value Proposition** — one sentence that explains why someone should choose you. Test: would a friend understand what you do after reading only this?
4. **Solution** — map features directly to problems. If a feature does not solve a listed problem, drop it.
5. **Channels** — how will you reach these customers? Be honest about what you can actually execute.
6. **Revenue Streams** — how do you make money? Price point?
7. **Cost Structure** — what does it cost to run this?
8. **Key Metrics** — what 3-5 numbers would tell you the business is working?
9. **Unfair Advantage** — something that cannot be easily copied or bought. It is OK to leave this blank initially — most startups do not have one yet. But you need a plan to build one.

### When to Update
Update the canvas whenever a key assumption is validated or invalidated. A static Lean Canvas is as useless as a static business plan.

## Value Proposition Canvas

Best for: Understanding product-market fit. Zooms in on the connection between what you offer and what customers need.

### Customer Profile (right side)

**Customer Jobs** — What is the customer trying to get done?
- Functional jobs (tasks, problems to solve)
- Social jobs (how they want to be perceived)
- Emotional jobs (how they want to feel)

**Pains** — What prevents or makes it difficult to get the job done?
- Undesired outcomes or problems
- Obstacles
- Risks

**Gains** — What outcomes and benefits do customers want?
- Required gains (basic expectations)
- Expected gains (standard expectations)
- Desired gains (above expectations)
- Unexpected gains (delightful surprises)

### Value Map (left side)

**Products & Services** — What do you offer?
- Physical/tangible
- Digital/intangible
- Financial
- Intangible (expertise, status)

**Pain Relievers** — How do your products/services eliminate or reduce customer pains?
Map each pain reliever to a specific pain.

**Gain Creators** — How do your products/services create customer gains?
Map each gain creator to a specific gain.

### Fit Assessment

- **Problem-Solution Fit**: Do your pain relievers and gain creators address the jobs, pains, and gains that customers care about most?
- **Product-Market Fit**: Do customers validate that your solution works? (Evidence: willingness to pay, retention, referrals)

Rate each connection: Strong / Moderate / Weak / No fit.

## Business Model Canvas

Best for: Established businesses, comprehensive business model mapping. More detailed than Lean Canvas.

```

| KEY PARTNERS      | KEY ACTIVITIES    | VALUE             | CUSTOMER          | CUSTOMER          |
|                   |                   | PROPOSITIONS      | RELATIONSHIPS     | SEGMENTS          |
| Who are key       | What key things   |                   |                   |                   |
| partners and      | must we do to     | What value do     | What relationship | Who are our most  |
| suppliers?        | deliver value?    | we deliver to     | does each segment | important         |
|                   |                   | the customer?     | expect?           | customers?        |
+-------------------+-------------------+                   +-------------------+-------------------+
| KEY RESOURCES     |                   |                   | CHANNELS          |                   |
|                   |                   |                   |                   |                   |
| What key          |                   |                   | How do we reach   |                   |
| resources does    |                   |                   | and deliver to    |                   |
| our value prop    |                   |                   | customers?        |                   |
| require?          |                   |                   |                   |                   |
+-------------------+-------------------+-------------------+-------------------+-------------------+
| COST STRUCTURE                                            | REVENUE STREAMS                       |
|                                                           |                                       |
| What are the most important costs?                        | For what value do customers pay?      |

```

Fill all 9 blocks. The canvas is useful for identifying disconnects — e.g., if your channels do not reach your segments, or your key activities do not support your value proposition.

## Blue Ocean Strategy Tools

Best for: Finding uncontested market space. Useful when the competitive landscape feels crowded.

### Strategy Canvas

Plot competing factors on the X-axis and the level of offering on the Y-axis. Draw curves for your business and competitors.

```
High |          *
     |    *    / \        *
     |   / \  /   \      / \
     |  /   \/     \    /   \
     | /              \/     \
Low  |/                       \
     +--+--+--+--+--+--+--+--+
     F1  F2  F3  F4  F5  F6  F7  ← Competing Factors

--- Competitor A    --- Competitor B    --- Your Strategy
```

The goal: your curve should look fundamentally different from competitors, not just slightly higher on every factor.

### Four Actions Framework (ERRC Grid)

| Action | Question | Factors |
|--------|----------|---------|
| **Eliminate** | Which factors that the industry takes for granted should be eliminated? | |
| **Reduce** | Which factors should be reduced well below the industry standard? | |
| **Raise** | Which factors should be raised well above the industry standard? | |
| **Create** | Which factors should be created that the industry has never offered? | |

This forces you to break the value-cost trade-off instead of just trying to be better on every dimension.

### Three Tiers of Noncustomers

1. **Soon-to-be noncustomers** — On the edge of your market, minimally using the current solution.
2. **Refusing noncustomers** — Consciously chose against your market's offerings.
3. **Unexplored noncustomers** — In distant markets, never considered your industry's offerings.

Each tier reveals different opportunities for market expansion.

## Jobs-to-be-Done

Best for: Understanding what customers really need (vs. what they say they want).

### Core Framework

People do not buy products. They "hire" products to get a job done.

**Job Statement Format:**
```
When [situation/context],
I want to [motivation/job],
so I can [expected outcome].
```

**Example:**
```
When I'm commuting and have 20 minutes to kill,
I want to catch up on industry news,
so I can sound informed in meetings.
```

### Types of Jobs
- **Functional**: The practical task (get from A to B)
- **Emotional**: How the person wants to feel (feel safe, confident, in control)
- **Social**: How they want to be perceived (professional, innovative, responsible)

### Applying JTBD to Business Planning
1. Interview customers. Ask about the last time they [did the job], not what features they want.
2. Map the full job: what happens before, during, and after?
3. Identify where the current solutions fail (overserved and underserved steps).
4. Design your solution around the underserved steps.

## OKR-Based Planning

Best for: Turning strategy into measurable execution. Bridges the gap between a business plan and daily work.

### Structure

**Objective**: What do we want to achieve? (Qualitative, inspiring, time-bound)
**Key Results**: How do we know we achieved it? (Quantitative, measurable, 3-5 per objective)

### Business Plan OKRs (Example)

**Objective 1: Validate product-market fit by Q2**
- KR1: Achieve 40% week-over-week retention for 4 consecutive weeks
- KR2: Get 10 customers to $100+ MRR without discounting
- KR3: Net Promoter Score > 50 from first 50 users

**Objective 2: Build a repeatable acquisition engine by Q3**
- KR1: CAC < $50 from at least 2 channels
- KR2: 500 signups/month with < $2,000/month spend
- KR3: Organic traffic > 30% of total signups

**Objective 3: Reach break-even by Q4**
- KR1: MRR > $15,000
- KR2: Gross margin > 70%
- KR3: Monthly burn rate < $3,000

### OKR Quality Rules
- Key results must be measurable with a number, not subjective
- 70% achievement is the target (if you always hit 100%, goals are too easy)
- Review and adjust quarterly
- OKRs are not tasks — they are outcomes

## Desirability-Feasibility-Viability

Best for: Quick go/no-go assessment of a business idea. Simple but powerful screening tool.

| Dimension | Question | Evidence Needed |
|-----------|----------|-----------------|
| **Desirability** | Do customers actually want this? | Interviews, waitlist, LOIs, search volume, competitor traction |
| **Feasibility** | Can we actually build and deliver this? | Technical assessment, team capabilities, resource requirements |
| **Viability** | Can this be a sustainable business? | Unit economics, market size, competitive dynamics, path to profit |

A viable business requires all three. Missing any one is a fatal flaw:
- Desirable + Feasible but not Viable = hobby, not a business
- Desirable + Viable but not Feasible = great idea you cannot execute
- Feasible + Viable but not Desirable = a solution looking for a problem

## Framework Selection Guide

| Situation | Recommended Framework |
|-----------|-----------------------|
| "I have a vague idea" | Lean Canvas → DFV check |
| "I need to understand my customer" | Jobs-to-be-Done → Value Proposition Canvas |
| "My market feels crowded" | Blue Ocean (Strategy Canvas + ERRC) |
| "I need a comprehensive business model" | Business Model Canvas |
| "I need to turn strategy into action" | OKR-Based Planning |
| "Should I pursue this idea?" | DFV check → Lean Canvas |
| "How do I differentiate?" | Strategy Canvas → Competitive Positioning |
| "I need to validate assumptions" | Lean Canvas (fill and test systematically) |

Multiple frameworks can be combined. A common sequence for new ventures:
1. JTBD interviews → understand the real job
2. Lean Canvas → map the business on one page
3. DFV check → quick screen for fatal flaws
4. Value Proposition Canvas → deepen product-market fit
5. Full Business Plan → comprehensive document if needed
