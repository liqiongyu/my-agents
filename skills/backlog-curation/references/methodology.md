# Methodology

Backlog curation keeps durable work tracking clean after execution, review, and
learning.

Use these questions:

- Is this item now complete and closable?
- Is it stale or blocked enough that it should recycle?
- Does it overlap a better existing issue?
- Did execution generate follow-up work with a distinct acceptance boundary?
- What provenance should be preserved if we close, merge, or recycle it?
