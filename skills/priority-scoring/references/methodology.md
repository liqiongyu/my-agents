# Methodology

Priority scoring converts a shaped work item into a relative queue recommendation.

Use these five lenses:

- **Impact**: how broadly and materially does the issue affect users, revenue,
  correctness, or team throughput
- **Urgency**: how quickly does waiting make the problem worse
- **Reversibility**: how safe is deferral, and is there a reasonable workaround
- **Dependency pressure**: does this issue unblock other already-accepted work
- **Confidence**: how trustworthy is the current evidence behind the score

This skill should recommend pressure, not pretend to allocate budget or approve
execution.
